Spring 动态代理默认⽤哪⼀种

Spring 框架默认使用 基于接口的 JDK 动态代理1。如果目标对象实现了至少一个接口，则 Spring 会使用 JDK 动态代理。否则，Spring 将使用 CGLib 动态代理

CGLib 动态代理

CGLIB (Code Generation Library) 是一个基于 ASM 的字节码生成库，它允许我们在运行时对字节码进行修改和动态生成。CGLIB 动态代理是针对类来实现代理的，它的原理是对指定的目标类生成一个子类，并覆盖其中方法实现增强。相比 JDK 自带的动态代理，CGLIB 动态代理无需目标对象实现接口，因此可以为任何类创建代理对象。CGLIB 底层使用 ASM（一个 Java 字节码操纵框架）操作字节码数组来实现类代理。

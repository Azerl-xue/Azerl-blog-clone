Spring 动态代理默认⽤哪⼀种Spring 框架默认使用 基于接口的 JDK 动态代理1。如果目标对象实现了至少一个接口，则 Spring 会使用 JDK 动态代理。否则，Spring 将使用 CGLib 动态代理 CGLib 动态代理CGLIB (Code Generation Library) 是一个基于 ASM 的字节码生成库，它允许我们在运行时对字节码进行修改和动态生成。CGLIB 动态代理是针对类来实现代理的，它的原理是对指定的目标类生成一个子类，并覆盖其中方法实现增强。相比 JDK 自带的动态代理，CGLIB 动态代理无需目标对象实现接口，因此可以为任何类创建代理对象。CGLIB 底层使用 ASM（一个 Java 字节码操纵框架）操作字节码数组来实现类代理。
拦截器和过滤器了解么？
发表于2023-09-11|更新于2023-09-11|面试题|框架•Spring•面试题
拦截器和过滤器了解么？过滤器和拦截器都是 AOP（面向切面编程）的实现方式，但它们的底层实现方式不同。 过滤器是基于函数回调的，而拦截器则是基于 Java 的反射机制（动态代理）实现的。 这意味着过滤器依赖于 Servlet 容器，而拦截器不依赖于 Servlet 容器。 过滤器可以对几乎所有的请求起作用，包括 CSS、JS 等资源文件。它们可以在请求到达服务器之前或之后执行某些操作。过滤器通常用于处理请求参数、字符编码、日志记录等。 拦截器只对 action 请求起作用，即对外访问路径。一个应用中可以同时存在多个拦截器，一个请求也可以触发多个拦截器，而每个拦截器的调用会依据它的声明顺序依次执行。拦截器通常用于权限验证、日志记录等。 希望这能帮助你更好地理解拦截器和过滤器之间的区别。

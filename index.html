讲讲Spring框架的加载流程
发表于2023-09-13|更新于2023-09-13|面试题|Spring•面试题
讲讲Spring框架的加载流程 加载配置文件：Spring框架首先会加载指定的配置文件，如XML配置文件或Java配置类。 实例化BeanFactory：Spring框架会根据配置文件中定义的Bean信息，实例化BeanFactory，即IoC容器。 实例化Bean：Spring框架会根据BeanFactory中定义的Bean信息，实例化Bean对象，并将其存储到IoC容器中。 处理Bean依赖关系：Spring框架会分析Bean之间的依赖关系，并完成依赖注入，即将需要的Bean注入到其他Bean中。 初始化Bean：Spring框架会调用Bean的初始化方法，对Bean进行初始化操作。 提供Bean：Spring框架会将所有初始化完成的Bean提供给应用程序使用。 总的来说，Spring框架的加载流程主要包括配置文件加载、IoC容器实例化、Bean实例化、依赖注入、Bean初始化和Bean提供等几个关键步骤，这些步骤的顺序和具体实现方式会根据不同的应用场景和配置文件而有所不同。

Spring Boot ⽀持哪些嵌⼊式 web 容器？Spring Boot 支持以下嵌入式 Web 容器¹: Tomcat: Tomcat 是一个流行的开源 Web 服务器，由 Apache 软件基金会开发和维护。Spring Boot 默认使用 Tomcat 作为嵌入式 Web 容器¹. Spring Boot 的⾃动配置是如何实现的?Spring Boot 的自动配置是通过 @EnableAutoConfiguration 注解实现的¹. 这个注解可以帮助我们自动载入应用程序所需要的所有默认配置。它通过条件化配置实现，这意味着只有在特定条件下才会应用这些配置⁴. 在启动 Spring Boot 应用程序时，@EnableAutoConfiguration 注解会扫描 classpath 下的 META-INF/spring.factories 文件，将其包装成 Properties 对象，并从中获取 key 值为 EnableAutoConfiguration 的数据，然后添加到容器中²³. 这些配置信息是从 Spring Boot 的依赖关系和配置信息中获取的，它们可以帮助我们自动配置应用程序所需的 Spring Bean⁴.

什么是 Spring Boot Starter?Spring Boot Starter 是 Spring Boot 框架提供的一种便捷的依赖管理方式¹. 它是一组预定义的依赖项，可以帮助开发人员快速构建 Spring 应用程序². Starter 通常命名遵循的格式为 spring-boot-starter-{name}，例如 spring-boot-starter-data-mongodb². Spring Boot Starter 通过自动配置和自动装配来简化 Spring 应用程序的开发¹. 它提供了各种场景依赖启动器，并默认提供了各种自动化配置类。使用 Spring Boot Starter 开发项目时，一旦引入了某个场景的依赖启动器，Spring Boot 内部提供的默认自动化配置类就会生效，从而极大减少了开发人员的工作量².

hibernate 和 mybatis 区别
发表于2023-09-11|更新于2023-09-11|面试题|框架•Spring•面试题
hibernate 和 mybatis 区别Hibernate 和 MyBatis 都是 Java 中常用的 ORM（对象关系映射）框架，但它们的实现方式和使用场景略有不同. Hibernate 是一个全功能的 ORM 框架，它将 Java 类映射到数据库表，并提供了一种自定义查询语言（HQL 或 EJB QL） Hibernate 通过缓存、延迟初始化等技术实现高性能2. Hibernate 还提供了自动生成表的功能，无需手动编写 SQL 语句. MyBatis 是一个轻量级的持久化框架，它将 SQL 语句映射到 Java 方法 MyBatis 强调使用 SQL 语句，而不是像 Hibernate 等其他 ORM 框架一样使用自定义查询语言. MyBatis 可以与 Spring 框架集成. 总体而言，Hibernate 更适合处理复杂的业务逻辑和大型应用程序，而 MyBatis 更适合处理简单的 CRUD（增删改查）操作和小型应用程序。此外，Hibernate 支持基于接口和 CGLib 的动态代理，而 MyBatis 默认使用基于接口的 JDK 动态代理.

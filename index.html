Spring Boot ⽀持哪些嵌⼊式 web 容器？
发表于2023-09-11|更新于2023-09-11|面试题|框架•Spring•面试题•Spring Boot
Spring Boot ⽀持哪些嵌⼊式 web 容器？Spring Boot 支持以下嵌入式 Web 容器¹: Tomcat: Tomcat 是一个流行的开源 Web 服务器，由 Apache 软件基金会开发和维护。Spring Boot 默认使用 Tomcat 作为嵌入式 Web 容器¹. Jetty: Jetty 是另一个流行的开源 Web 服务器，由 Eclipse 基金会开发和维护。Spring Boot 也支持使用 Jetty 作为嵌入式 Web 容器¹. Undertow: Undertow 是一个轻量级的 Web 服务器，由 JBoss 开发和维护。Spring Boot 也支持使用 Undertow 作为嵌入式 Web 容器¹. Netty: Netty 是一个高性能的异步事件驱动的网络应用程序框架，由 JBoss 开发和维护。Spring Boot 也支持使用 Netty 作为嵌入式 Web 容器¹.

发表于2023-09-11|更新于2023-09-11|面试题|框架•Spring•面试题•Spring Boot
Spring Boot 的⾃动配置是如何实现的?Spring Boot 的自动配置是通过 @EnableAutoConfiguration 注解实现的¹. 这个注解可以帮助我们自动载入应用程序所需要的所有默认配置。它通过条件化配置实现，这意味着只有在特定条件下才会应用这些配置⁴. 在启动 Spring Boot 应用程序时，@EnableAutoConfiguration 注解会扫描 classpath 下的 META-INF/spring.factories 文件，将其包装成 Properties 对象，并从中获取 key 值为 EnableAutoConfiguration 的数据，然后添加到容器中²³. 这些配置信息是从 Spring Boot 的依赖关系和配置信息中获取的，它们可以帮助我们自动配置应用程序所需的 Spring Bean⁴.

什么是 Spring Boot Starter?Spring Boot Starter 是 Spring Boot 框架提供的一种便捷的依赖管理方式¹. 它是一组预定义的依赖项，可以帮助开发人员快速构建 Spring 应用程序². Starter 通常命名遵循的格式为 spring-boot-starter-{name}，例如 spring-boot-starter-data-mongodb². Spring Boot Starter 通过自动配置和自动装配来简化 Spring 应用程序的开发¹. 它提供了各种场景依赖启动器，并默认提供了各种自动化配置类。使用 Spring Boot Starter 开发项目时，一旦引入了某个场景的依赖启动器，Spring Boot 内部提供的默认自动化配置类就会生效，从而极大减少了开发人员的工作量².

hibernate 和 mybatis 区别
发表于2023-09-11|更新于2023-09-11|面试题|框架•Spring•面试题
hibernate 和 mybatis 区别Hibernate 和 MyBatis 都是 Java 中常用的 ORM（对象关系映射）框架，但它们的实现方式和使用场景略有不同. Hibernate 是一个全功能的 ORM 框架，它将 Java 类映射到数据库表，并提供了一种自定义查询语言（HQL 或 EJB QL） Hibernate 通过缓存、延迟初始化等技术实现高性能2. Hibernate 还提供了自动生成表的功能，无需手动编写 SQL 语句. MyBatis 是一个轻量级的持久化框架，它将 SQL 语句映射到 Java 方法 MyBatis 强调使用 SQL 语句，而不是像 Hibernate 等其他 ORM 框架一样使用自定义查询语言. MyBatis 可以与 Spring 框架集成. 总体而言，Hibernate 更适合处理复杂的业务逻辑和大型应用程序，而 MyBatis 更适合处理简单的 CRUD（增删改查）操作和小型应用程序。此外，Hibernate 支持基于接口和 CGLib 的动态代理，而 MyBatis 默认使用基于接口的 JDK 动态代理.
Spring 动态代理默认⽤哪⼀种
发表于2023-09-11|更新于2023-09-11|面试题|框架•Spring•面试题
Spring 动态代理默认⽤哪⼀种Spring 框架默认使用 基于接口的 JDK 动态代理1。如果目标对象实现了至少一个接口，则 Spring 会使用 JDK 动态代理。否则，Spring 将使用 CGLib 动态代理 CGLib 动态代理CGLIB (Code Generation Library) 是一个基于 ASM 的字节码生成库，它允许我们在运行时对字节码进行修改和动态生成。CGLIB 动态代理是针对类来实现代理的，它的原理是对指定的目标类生成一个子类，并覆盖其中方法实现增强。相比 JDK 自带的动态代理，CGLIB 动态代理无需目标对象实现接口，因此可以为任何类创建代理对象。CGLIB 底层使用 ASM（一个 Java 字节码操纵框架）操作字节码数组来实现类代理。
拦截器和过滤器了解么？
发表于2023-09-11|更新于2023-09-11|面试题|框架•Spring•面试题
拦截器和过滤器了解么？过滤器和拦截器都是 AOP（面向切面编程）的实现方式，但它们的底层实现方式不同。 过滤器是基于函数回调的，而拦截器则是基于 Java 的反射机制（动态代理）实现的。 这意味着过滤器依赖于 Servlet 容器，而拦截器不依赖于 Servlet 容器。 过滤器可以对几乎所有的请求起作用，包括 CSS、JS 等资源文件。它们可以在请求到达服务器之前或之后执行某些操作。过滤器通常用于处理请求参数、字符编码、日志记录等。 拦截器只对 action 请求起作用，即对外访问路径。一个应用中可以同时存在多个拦截器，一个请求也可以触发多个拦截器，而每个拦截器的调用会依据它的声明顺序依次执行。拦截器通常用于权限验证、日志记录等。 希望这能帮助你更好地理解拦截器和过滤器之间的区别。
